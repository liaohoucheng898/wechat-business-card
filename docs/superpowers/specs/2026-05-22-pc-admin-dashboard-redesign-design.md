# PC 端后台强数据驾驶舱改版设计规格

## 背景

当前 PC 管理端是 Vue 3 + Element Plus 后台，已经包含登录、首次改密、员工管理、公司管理、案例管理、案例编辑、公司编辑和数据统计。现有界面更接近基础后台：白色侧边栏、普通卡片、列表表格和简单弹窗。用户已确认采用“强数据驾驶舱”方向，先以运营管理中心组织全套后台，再把具体管理模块放在二级业务入口中。

本规格只描述真实程序修改目标。已确认的静态原型文件为：

- `项目文档/原型设计/2026-05-22_PC端后台强数据驾驶舱原型_V1.html`

## 用户与任务判断

主要用户是企业管理员。管理员使用频率高，任务包括查看运营数据、维护员工账号、维护公司资料、维护客户案例、检查小程序展示内容、追踪导入和重置密码等高风险操作。

成功标准是：管理员进入后台后，能先看清当前业务状态、异常风险和待处理事项，再快速进入具体管理页面完成编辑、导入、删除、重置密码、预览和统计分析。

失败代价包括：误删案例、误停用员工、错误重置密码、覆盖他人编辑、漏看无栏目案例、误解统计口径、把未完整维护的公司资料发布到小程序端。

## 设计方向

采用强数据驾驶舱路线。后台默认首页从“案例管理”调整为“运营驾驶舱”，但现有业务模块不另起平行实现，仍复用当前真实生效的页面、接口、云函数和配置。

视觉遵循 `E:\Codex\shared-huayue\design-huayue.md`：

- Web 端使用左侧深色侧边导航、顶部栏和内容区。
- 页面背景为浅灰，内容面板为白色边框面板。
- 表格页优先高密度、筛选、工具栏、分页和批量处理。
- 普通卡片不用阴影制造高级感，主要靠边框、间距和层级。
- 主按钮每个操作区最多一个。
- 状态标签必须颜色加文字。
- 指标和报表必须有来源、统计口径和更新时间。
- 高风险操作必须说明对象、范围、后果和是否可撤回。

## 信息架构

左侧导航改为以下分组：

1. 入口与总览
   - 运营驾驶舱
   - 登录页
   - 首次改密
2. 业务模块
   - 内容中心
   - 人员管理
   - 公司管理
   - 数据分析

真实程序中登录页和首次改密页不显示在已登录后的左侧导航中。它们只作为独立入口页面重设计。左侧导航实际显示：

- 运营驾驶舱
- 内容中心
- 人员管理
- 公司管理
- 数据分析

## 页面范围

### 登录页

登录页保留管理员手机号加密码登录。页面需要说明临时锁定规则、忘记密码处理路径和后台用途。登录失败提示要说明原因和下一步，不只显示“登录失败”。

### 首次修改密码页

首次临时密码登录后，仍强制进入修改密码页。页面说明新密码要求和修改成功后的去向。保存成功后进入运营驾驶舱。

### 全局布局

`AdminLayout.vue` 改为深色侧边栏、白色顶部栏和浅灰内容区。顶部栏承载当前面包屑、全局搜索占位、帮助入口、管理员信息和退出按钮。侧边栏展开宽度使用 224-232px，收起状态保留现有能力。

### 运营驾驶舱

新增后台默认首页。驾驶舱展示：

- 期间浏览量
- 活跃员工
- 可见案例
- 待处理风险
- 浏览趋势
- 风险与待处理列表
- 员工表现排行
- 最近操作

第一版可以使用现有接口和前端派生数据组合：统计数据来自 `adminGetStats`，案例风险来自 `adminGetCaseList`，员工风险来自 `adminGetStaffList`。如果某类数据暂时无法低成本获取，页面显示可解释的空状态或延迟状态，不新增复杂后端聚合。

### 内容中心

案例管理从“案例管理”命名升级为“内容中心”，但路由和接口可以继续复用 `/cases`。页面结构为标题区、主操作、公司/栏目/状态/关键词筛选、工具栏、表格和分页。保留栏目管理、案例预览、编辑、删除、可见性切换。

栏目管理弹窗要保留公司切换，栏目列表用表格维护名称、排序和案例数。删除栏目需说明影响范围。

### 案例编辑

案例编辑页保留现有字段：所属公司、企业全称、封面图、简要描述、排序值、所属栏目、是否可见、案例详情。页面改为主表单加右侧预览摘要。栏目仍按公司分组显示，避免同名栏目混淆。页面显示草稿/未保存状态，并保留保存、取消和预览操作。

富文本相关修改必须同步案例编辑页预览、案例列表预览、小程序展示链路和公司富文本场景，不能只改一个入口。

### 人员管理

员工管理保留新增、导入、编辑、重置密码、停用、启用、删除。页面增加搜索和筛选区，表格信息层级优化：员工、手机号、职位、已绑定微信数、密码状态、账号状态、操作。员工编辑继续优先使用抽屉，不改数据结构。

导入员工弹窗要说明模板字段、自动开通规则、文件限制、公式和重复手机号校验。导入结果需要显示总数、成功数、失败数和失败明细。

重置密码弹窗要明确旧密码立即失效，结果弹窗展示账号和新临时密码。

### 公司管理

公司管理从卡片列表改为业务配置表，更适合维护固定三家公司资料。表格展示公司、资料完整度、定位、联系电话、更新时间和编辑入口。公司编辑页保留公司名称、Logo、简介、业务介绍、地址、电话、定位、官网。富文本编辑器样式与案例编辑保持一致。

### 数据分析

数据统计页升级为“数据分析”。页面先给统计口径和更新时间，再展示期间浏览量、累计浏览量、高活跃员工、趋势图和员工排行榜。保留公司切换和时间范围切换。

统计页必须说明：只统计外部访客打开名片产生的 `card_view` 日志，已登录员工访问不计入。指标卡和图表不得只显示数字或图形。

### 状态与风险

真实程序至少覆盖以下状态：

- 空状态：没有员工、没有案例、筛选无结果。
- 权限不足：按钮禁用时说明不能做什么、为什么、找谁授权。
- 数据冲突：已有 `pageLoadedAt` 并发保护的页面要把冲突原因说清楚。
- 部分成功：导入员工失败明细。
- 高风险确认：删除、停用、重置密码、隐藏案例。
- 加载中和错误：表格和统计请求失败时页面内说明，不只依赖短暂消息。

## 技术约束

- 技术栈继续使用 Vue 3、Vue Router、Pinia、Element Plus、ECharts、TinyMCE。
- 不替换 UI 组件库，不引入新图表库。
- 不改数据库结构。
- 不改云函数接口，除非实现驾驶舱时发现现有数据无法以合理成本获取。
- 优先复用当前真实页面、接口和组件，不另做一套平行后台。
- Windows 下命令优先使用 `.cmd` 或 `.exe`，避免调用 `.ps1`。

## 验收标准

- 本地构建通过：`cmd /c npm.cmd --prefix admin run build`。
- 后台默认登录后进入运营驾驶舱。
- 左侧导航、顶部栏、内容区符合华悦规范。
- 登录页、首次改密页、运营驾驶舱、内容中心、案例编辑、人员管理、公司管理、公司编辑、数据分析均能打开。
- 表格页有筛选区、工具栏、表格、状态标签和空/错误状态。
- 高风险操作确认文案说明对象、后果和可撤回性。
- 统计指标显示数据来源、统计口径和更新时间。
- 390px 宽度下核心内容可读，页面本身不出现横向滚动；宽表只在表格容器内横向滚动。
- Codex 可执行的构建、静态检查和浏览器渲染验证必须先完成，再交给用户人工确认。

## 人工验证建议

必须人工验证：

- 管理员真实登录后默认进入运营驾驶舱。
- 新增/编辑案例保存后，小程序端展示不受影响。
- 公司简介、业务介绍和案例详情的富文本在小程序端显示正常。
- 员工重置密码后，员工能用新临时密码完成首次绑定。

建议人工验证：

- 不同公司筛选、栏目管理、案例预览是否符合日常使用习惯。
- 统计数据与真实访问日志口径是否符合预期。
- 常用电脑屏幕下表格密度和文字大小是否舒适。

# 发布记录

## 使用规则

1. 每次上线前，先执行一次本地备份脚本，生成本次发布包。
2. 每次上线后，在本文件末尾追加一条发布记录。
3. 发布记录统一使用分段方式书写，不使用单行表格。
4. 如果线上有问题，优先按对应批次的备份目录回滚。

## 备份脚本

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\create-release-backup.ps1 -ReleaseName "本次发布说明"
```

默认会在 `release_backups/时间戳_发布说明/` 下生成：

- `admin-dist.zip`
- `cloudfunction-adminGetStaffList.zip`
- `RELEASE.txt`

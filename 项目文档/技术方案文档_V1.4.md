### 1.2 技术选型

| 层 | 技术 | 版本 | 选择理由 |
|----|------|------|---------|
| **小程序端** | 微信原生框架 | 基础库 ≥ 2.25.0 | 20人团队内部工具，不需要跨端，原生性能最好、包体最小 |

| **图片裁剪** | vue-cropper | — | PC端头像/封面裁剪 |

## 二、项目目录结构

### 2.1 小程序端
```
miniprogram/
├── app.js                      # 小程序启动入口，负责默认启动分流、静默登录、全局分享、全局错误处理
├── app.json                    # 页面注册与全局窗口配置
├── app.wxss                    # 全局样式
├── sitemap.json                # 小程序搜索配置
├── utils/
│   ├── auth.js                 # sessionToken 本地存取、过期判断、清理
│   ├── cloud.js                # 云函数调用封装
│   ├── format.js               # 格式化工具
│   └── toast.js                # 统一 Toast / Modal 封装
├── components/
│   ├── empty-state/            # 空状态组件
│   └── phone-verify/           # 旧验证码表单组件（当前页面未引用，保留兼容）
├── pages/
│   ├── card/                   # M1 名片展示页（访客）
│   ├── card-disabled/          # M2 名片已注销页
│   ├── bind-phone/             # M3 账号密码绑定页（首次绑定当前微信）
│   ├── my-card/                # M4 我的名片页
│   ├── edit-card/              # M5 编辑个人名片页
│   ├── my-stats/               # M7 个人数据统计页
│   └── case-detail/            # 客户案例详情页
├── images/                     # 默认头像、空状态、系统 Logo 等静态资源
└── cloudfunctions/             # 云函数目录（见 §2.3）
```

  > 当前源码已包含独立的 `case-detail` 页面；客户从名片页点击案例卡片后，会跳转至案例详情页并通过 `getCaseDetail` 拉取完整内容。
  > 当前源码中也没有 `card-preview`、`company-tab`、`time-tab` 这类小程序自定义组件，对应能力分别在页面内和 PC 端组件中实现。
  > 当前源码中，`bind-phone/` 直接承担员工首次账号密码绑定能力，验证码绑定入口已下线。
  > 当前源码中，`app.js` 会在“小程序图标默认启动”场景下读取 `launchOptions`：已绑定员工默认跳转 `my-card`，未绑定员工跳转 `bind-phone`；带 `staffId/companyId` 的分享链接场景保持 `card` 页访客链路。

## 四、云函数接口清单

> 完整的出入参定义。PRD中的接口清单为概要版本，此处为开发实现的权威版本。
> 本章接口说明已按当前源码同步；如后续继续调整登录链路或下线遗留兼容入口，需要再次同步更新。

### 4.0 通用规范

**统一响应格式：**

```json
// 成功
{ "code": 0, "data": { ... }, "msg": "ok" }

// 失败
{ "code": "E0201", "data": null, "msg": "验证码已过期，请重新获取" }
```

**错误码编号规则：** `E` + 模块编号(2位) + 序号(2位)

| 模块编号 | 模块 |
|---------|------|
| 01 | 通用（参数/服务器/限流/并发冲突） |
| 02 | 认证（登录/绑定/Session/短信） |
| 03 | 员工 |
| 04 | 公司 |
| 05 | 案例 |
| 06 | 统计 |
| 07 | 文件 |

**Token校验逻辑（`_shared/auth.js`）：**

```
普通员工接口（序号5-8）：
  → 从入参取 sessionToken
  → 查 staff 表：token匹配 + sessionExpireAt > now + status=active
  → 通过 → 注入 staffInfo 到 context
  → 失败 → 返回 E0207

管理员接口（序号9-25）：
  → `@cloudbase/node-sdk` 从当前 Web SDK 登录态中读取 `customUserId`（即 `staffId`）
  → 按 `staffId` 读取 staff 表，校验 status=active
  → 校验 isAdmin=true
  → 查 admin_config：phone ∈ adminPhones
  → 通过后注入管理员 staffInfo
  → 任一失败 → 返回 E0207 或 E0208
```

### 4.1 `login` — 小程序登录

```
触发：小程序启动时
鉴权：无（获取openid由云函数自动完成）

入参：无（openid从wx.cloud上下文自动获取）

逻辑：
  1. 获取 OPENID
  2. 优先查 staff 表 where openids 包含 OPENID；未命中时回退查旧字段 openid = OPENID
  3. 未找到 → 返回 { needBind: true }
  4. 找到 + status=disabled → 返回 E0211
  5. 找到 + status=active →
     a. 生成新 sessionToken（32位随机串）
     b. 设 sessionExpireAt = now + 7天
     c. 若当前 OPENID 不在员工 `openids` 列表中，则自动追加写入 `openids`
     d. 若旧字段 `openid` 为空，则回写首个 OPENID 作为兼容值
     e. 更新 staff 记录
     f. 返回 staffInfo + sessionToken（其中 avatar 为可直接展示的临时地址，avatarOriginal 保留原始文件ID）

出参（成功-已绑定）：
{
  "code": 0,
  "data": {
    "needBind": false,
    "staffInfo": {
      "staffId": "xxx",
      "name": "张三",
      "phone": "13800001234",
      "wechat": "zhangsan_wx",
      "email": "zhang@huayue.com",
      "bio": "...",
      "avatar": "https://temp.example.com/avatar.jpg",
      "avatarOriginal": "cloud://xxx",
      "enabledCompanies": [...],
      "isAdmin": false
    },
    "sessionToken": "a1b2c3...",
    "sessionExpireAt": 1234567890
  }
}

出参（成功-未绑定）：
{ "code": 0, "data": { "needBind": true } }

可能错误码：E0102, E0211
```

### 4.2 `sendSmsCode` — 发送短信验证码

```
触发：手机号验证页/PC登录页点击"获取验证码"
鉴权：无

入参：
{
  "phone": "13800001234",      // 必填，11位手机号
  "scene": "bind" | "login" | "changePhone"  // 必填，场景标识

逻辑：
  1. 校验 phone 格式（11位，1开头）
  2. 频率限制检查：
     a. 同一手机号60秒内不能重发 → E0203
       b. 同一手机号每日不超过20条 → E0204
       c. 同一设备（IP/openid）每日不超过20条 → E0205
  3. 生成6位随机验证码
  4. 调用腾讯云SMS发送
  5. 将验证码+过期时间(5分钟)存入临时表/缓存
```

### 4.3 `bindPhone` — 绑定手机号

```

{
  "phone": "13800001234",
  "smsCode": "123456"

逻辑：
  1. 校验验证码（匹配+未过期+未锁定）
  2. 连续错误5次 → 锁定15分钟 → E0206
  3. 查 staff 表 where phone = 入参phone
  4. 未找到 → E0209
  5. 找到 + status=disabled → E0211
  6. 找到 + openid已有值且≠当前openid → E0210

     a. 写入 openid
     b. 如果 isAdmin=true → 将openid加入 admin_config.adminOpenids
  ```

### 4.3A `passwordLogin` — 小程序账号密码绑定

```text
  3. 合并目标员工旧 `openid` 与新 `openids`，得到去重后的已绑定微信列表
  4. 从列表中移除 `targetOpenid`
  5. 更新目标员工：
     - `openids = nextOpenids`
     - `openid = nextOpenids[0] || ''`
     - `wechatBindings = nextWechatBindings`
     - `updatedAt = now`
  6. 如果目标员工是管理员，则从 `admin_config.adminOpenids` 中同步移除 `targetOpenid`
  7. 返回解绑后的绑定数量和绑定列表

出参：
{
  "code": 0,
  "data": {
    "boundWechatCount": 1,
    "boundWechatOpenids": ["openid-b"],
    "boundWechatBindings": [
      { "openid": "openid-b", "remark": "备用微信" }
    ]
  }
}

可能错误码：E0101, E0102, E0208, E0302
```

> 解绑只解除“某一个微信号”和员工账号的关系，不删除员工档案、手机号、职位、头像、统计数据，也不影响该员工其他已绑定微信继续使用。

## 六、认证与安全

### 6.1 Session管理

| 端 | Token字段 | 过期字段 | 有效期 | 存储位置(客户端) |
|----|-----------|---------|--------|-----------------|
| 小程序 | `sessionToken` | `sessionExpireAt` | 7天 | `wx.setStorageSync` |
| PC端 | CloudBase SDK 登录态 | SDK 内部维护 | 由 CloudBase 控制 | SDK 持久化 + `localStorage.adminInfo` |

- 两端Token互相独立，互不影响
- Token格式：32位随机字符串（`crypto.randomBytes(16).toString('hex')`）
- 每次登录生成新Token，旧Token自动失效（单设备登录）
- 若 `mustChangePassword = true`，当前主要用于管理员 PC 后台临时密码改密链路；员工端当前不会再因该字段强制跳转

### 6.2 短信安全策略

| 维度 | 限制 |
|------|------|
| 同一手机号发送间隔 | ≥ 60秒 |
| 同一手机号日发送量 | ≤ 20条 |
| 同一设备(IP/openid)日发送量 | ≤ 20条 |
| 验证码有效期 | 5分钟 |
| 连续错误 | 5次后锁定15分钟 |
| 验证码长度 | 6位纯数字 |

### 6.3 账号密码安全策略（V1.6 规划）

| 项目 | 方案 |
|------|------|
| 账号 | 统一使用 `staff.phone` |
| 密码存储 | `crypto.scrypt` + 随机 `salt`，只存 `passwordHash` |
| 初始密码 | 系统生成 8 位临时密码，排除 `0/O/o`、`1/l/I` 等易混淆字符 |
| 首次登录 | 员工端首次绑定后当前直接进入业务页；管理员使用临时密码登录后台时仍需先修改密码 |
| 输错锁定 | 连续输错 5 次，锁定 15 分钟 |
| 日志要求 | 不打印明文密码、验证码、`OPENID`、`sessionToken`，也不打印完整哈希和整包敏感结果 |
| 管理员重置 | 重置后旧密码立即失效，新临时密码只在响应弹窗里展示一次 |

### 6.4 并发编辑保护

```
编辑流程：
  1. 进入编辑页 → 记录当前 updatedAt 为 pageLoadedAt
  2. 保存时 → 入参携带 pageLoadedAt
  3. 云函数 → 查 DB 中 updatedAt
     - 若 DB.updatedAt > pageLoadedAt → 返回 E0104（并发冲突）
     - 若 DB.updatedAt ≤ pageLoadedAt → 正常更新 + 刷新 updatedAt
```

### 7.6 账号密码登录上线迁移

```text
适用场景：当前系统里已存在员工账号，但还没有密码字段

  1. 备份 staff / admin_config 集合
```
